# ASG 생성하면서 발생한 문제들

## 1. user data 없이 런치 템플릿 생성

### 장애 현상
ASG 생성 후 테스트하기 위해 기존에 있던 서버를 삭제하고, 새로 서버 생성하려고 할 때 계속 실패하면서 계속해서 새로운 인스턴스를 만들려고 시도함

### 장애 원인
런치 템플릿 생성할 때 user data에서 nginx 설치 스크립트가 없어서 nginx 설치되지 않은 채로 ec2가 생성됨.
ALB에서 healthy check 할 때 계속 unhealthhy 상태가 됨

### 장애 해결
user data에 nginx 설치 스크립트를 추가


## 2. ASG의 target group을 ALB의 미연동

### 장애 현상
1. ASG 생성할 때 신규로 만든 target group의 대상이 "unused" 상태로 보임
<p align="left">
  <img src="./images/asg-tg-issue-2.png" width="500">
</p>

2. target group의 healthy 기준은 http(80), 200 
<p align="left">
  <img src="./images/asg-issue2-healthy.png" width="500">
</p>

3. 각 인스턴스의 nginx는 running 상태

### 장애 원인

ASG의 target group이 ALB에 연동되지 않아서 target group에서 unused 상태가 됨.
ALB에는 기존에 만든 target group이 붙어있는 상태.
ALB 생성할 때 신규 target group을 선택해주면 로드밸런서와 연결되지 않았다고 뜸.

<p align="left">
  <img src="./images/asg-alb-target-group.png" width="500">
</p>

### 장애 해결
ASG 생성이 완료되면, ALB 리스너에서 기존 target group을 지우고 ASG target group 으로 변경해야함.
<p align="left">
  <img src="./images/alb-target-change.png" width="500">
</p>

변경하면 target group의 대상도 healthy로 정상 동작
<p align="left">
  <img src="./images/asg-issue2-solution.png" width="500">
</p>
