# docker image, container

## docker 개념
### docker 란?
- 컨테이너 기반의 오픈소스 가상화 플랫폼
- 즉, 애플리케이션을 독립된 환경에서 실행할 수 있게 해주는 도구
- 내 개발 환경을 통으로 말아서 전달할 수 있게 해주는 도구
- 아파트 안에서 원룸을 만들어주는 관리인.
- 커널은 공유하되, 파일 시스템과 라이브러리는 분리


### 가상머신과의 차이
가상머신:  독립된 운영체제를 가지고 있어 무겁고 오래걸림. 별도 커널이 있음.
도커 컨테이너 :  운영체재를 따로 가지고 있지 않고, 호스트 운영체제의 커널을 공유함.
              따라서 host에서 프로세스가 돌아감. 리눅스의 프로세스를 격리해서 실행하는 것.
              -> host에서 ps -ef 하면 container 안에서 돌아가는 프로세스가 잡힘. 
                 가상머신처럼 vm 내부적으로 별도 pid가 있지 않음.

### 이미지와 컨테이너의 차이
- 이미지 : 컨테이너를 만들기 위한 설계도/템플릿. 소스 코드, 라이브러리, 환경 설정 등 모두 들어가있음.
         이미지는 한번 만들어지면 수정이 안되기 때문에, 수정하려면 이미지를 새로 만들어야함. (read-only)
- 컨테이너 : 이미지를 실행한 것. 설계도를 가지고 지은 실제 집. 
-          -> 설계도 하나로 여러 집을 지을 수 있듯, 하나의 이미지로 여러 컨테이너를 실행할 수 있음.
           독립적인 실행 환경. 하나의 컨테이너가 망가져도 다른 컨테이너의 영향을 주지 않음.
           수정이 가능 (writable). 컨테이너 안에서 파일을 만들고 수정 가능.
           host의 운영체제를 사용하지 않고, 이미지 안에 들어있는 user space를 사용.
           

| 구분 | 이미지(Image) | 컨테이너(Container) |
|------|---------------|---------------------|
| 불변성 | 읽기 전용, 변경 불가능 | 쓰기 가능, 변경 가능 |
| 생명주기 | 삭제 전까지 영구적으로 존재| 시작, 중단, 삭제 생명주기 있음 |
| 저장 방식 | 레이어 형태로 효율적으로 저장 | 이미지 위에 쓰기 가능한 레이어 추가 |
| 용도 | 배포와 공유를 위한 템플릿, 설계도 | 실제 애플리케이션이 실행되는 환경 |
| 명령어 | images, pull, build| ps, run, stop |

### docker 호출 흐름
- docker.sock : docker cli(명령을 전달하는 클라이언트)와 docker Docker Daemon(dockerd)을 연결하는 통신 통로
                해당 파일의 권한은 srw-rw----
- Docker Daemon(dockerd) : 실제로 Docker Engine의 작업(이미지 관리, 컨테이너 생성)을 수행
- - Docker Hub :  이미지를 가져오는 외부 저장소, 이미지를 공유하는 저장소. github처럼 이미지를 올려놓는 곳. 
```bash
# docker 명령어 사용시 flow
ubuntu 사용자
      |
      | docker 명령어
      ↓
/var/run/docker.sock
      |
      ↓
Docker Daemon (dockerd)ㄴ
      |
      ↓
Docker Hub
```
